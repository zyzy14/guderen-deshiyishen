七步成馔

  1. 壹

    报上食材

    在输入框写下冰箱现有物，如"鸡胸肉、豆腐、大葱"，支持逗号、顿号、空格分隔；也能识别"鸡胸肉→鸡""大葱→葱"等古今异名。嫌麻烦？直接点页面上的示例更省事。

  2. 贰

    古法匹配

    系统用同义词归一 + 主料加权打分，从 22 道古籍食谱里挑出最合你手头食材的古方，并排好可翻页的"今夜食单"。

  3. 叁

    半文白秘籍

    为这道菜生成一段治愈系文案——开场引子点出你与古法的缘分，古法步骤让现代人能照做，再附《本草纲目》性味注、原典引文，与一句题跋收尾。

  4. 肆

    水墨成图

    程序化生成一碗水墨风成品图：青瓷 / 白瓷碗、笔触菜肴、袅袅热气、竖排菜名与一方红印章。无需任何绘图 API，离线即得。

  5. 伍

    考据小笺

    若你用了辣椒、番茄、土豆、胡萝卜等"穿越食材"，小笺会温柔提醒：辣椒是明万历才入中土，古人以花椒、姜取辛——长知识又不违和。

  6. 陆

    翻页与食单

    一道不合心意？左右翻页，或在"今夜食单"里切换，总有一味合你今夜的心境。

  7. 柒

    大模型润色（已接入）

    本店已接入 DeepSeek：你报上食材，开场引子、古法步骤与收尾题跋便由大模型实时重写，比离线模板更贴合今夜心境。想换服务商？在 .env 里改 LLM_PROVIDER 即可（通义千问 / OpenAI / 本地 Ollama 皆支持），界面随即亮起"✦ 大模型已接入"。

技术亮点

前端

纯静态 + 国风视觉：宣纸纹理、朱砂红、青瓷绿、毛笔字体，翻页动效。

知识库

recipes.json 单一数据源：22 道食谱、93 条古今同义词、9 条穿越考据。

匹配引擎

同义词归一 + 主料加权；弱匹配自动"变通"，取最清简的古法兜底。

文风生成

三档引子 + 结构化半文白模板，离线即用，亦可接大模型润色。

水墨图

SVG 程序化生成，按食谱主色与配料参数化，离线可得。

后端

Flask /api/cook：离线兜底 + 大模型 / AI 绘图接入位。

# 古人的深夜食堂 · 舌尖上的古籍

一个国风治愈系食谱生成器：输入冰箱里现成的食材，系统从四十余道古籍食谱中智能匹配最佳古方，生成水墨风菜品图、半文白引子与步骤、本草性味注、原典引文与题跋；并可接入大模型实时润色文案。

> 知识库考据自《齐民要术》《随园食单》《本草纲目》《山家清供》《粥谱》《云林堂饮食制度集》《东京梦华录》及苏轼、张翰等文人笔记。半文半白之辞为今人演绎，仅供下厨与把玩。

## 功能亮点

- **智能匹配**：同义词归一（如「鸡胸肉→鸡」「剩米饭→米」）+ 主料加权打分，从 46 道古籍食谱中挑出最合手头食材的古方。
- **治愈系文案**：开场引子、古法步骤、本草注、原典引文、收尾题跋，离线模板即可生成。
- **大模型润色**：接入 DeepSeek（亦支持通义千问 / OpenAI / 本地 Ollama），引子、步骤、题跋由大模型实时重写。
- **水墨菜品图**：纯 SVG 程序化生成（青瓷 / 白瓷碗、笔触菜肴、热气、竖排菜名、红印章），无需任何绘图 API。
- **灶台过场动画**：点击「为我备席」触发 SVG 动画——烧灶火→掀盖→起锅→暖光晕开→朱印落定。
- **考据小笺**：遇到辣椒、番茄、土豆等明清后才传入中国的「穿越食材」，温柔提示历史背景。
- **多页面**：古籍课堂（典籍时间线 + 来源筛选）、玩法导览、美食家人格测试、我的收藏。

## 技术栈

- 前端：纯静态 HTML / CSS / JS（无框架依赖），国风视觉体系，暗夜模式完整适配。
- 后端：Flask 提供 `/api/cook`（匹配引擎 + 大模型润色管道），离线兜底可用。
- 数据：单一数据源 `recipes.json` → 自动生成 `js/data.js`（`python3 build_data.py`）。
- 视觉生成：SVG 程序化（水墨图、灶台动画）。

## 本地运行

```bash
pip install flask requests
python3 server.py            # 默认 :5000
# 或指定端口
FLASK_PORT=8000 python3 server.py
```

打开 `http://localhost:<端口>/` 即可。

## 接入大模型（可选）

复制 `.env.example` 为 `.env`，填入密钥：

```ini
LLM_PROVIDER=deepseek
LLM_API_KEY=sk-你的key
```

重启服务后即进入「大模型润色」模式（界面亮起「✦ 大模型已接入」）。未填密钥则为离线模板模式，照常可用。

## 目录结构

```
index.html       首页（报食材 → 备席 → 今夜食单）
sources.html     古籍课堂
guide.html       玩法导览
quiz.html        美食家人格
favorites.html   我的收藏
welcome.html     关于本店
css/  js/        样式与脚本
recipes.json     单一数据源（食谱 / 同义词 / 穿越考据）
server.py        Flask 后端
build_data.py    由 recipes.json 生成 js/data.js
```

## 部署说明

- 前端为纯静态资源，可托管于任意静态服务 / GitHub Pages。
- `/api/cook` 与「大模型润色」依赖 Flask 后端；纯静态托管时前端会自动回退到本地匹配 + 模板文案（无 LLM 润色）。
